# Launch Requirements

**Location:** `/documents/launch-requirements.md`

---

## Purpose

Prioritized list of what's needed before accepting the first paying client (V1) vs what can wait (Later).

---

## V1 — Must Have

### Sales Assets
- [ ] Demo site with sample law firm content
- [ ] Loom demo script (3-5 min walkthrough)
- [x] One-page overview (see [[Pilot Offer]])

### Legal & Policy
- [ ] Website privacy policy
- [ ] Terms of service
- [ ] Standard disclaimer language
- [ ] Client service agreement template

### Technical Operations
- [ ] Hosting (Vercel, Railway, or similar)
- [ ] Per-firm data isolation
- [ ] Basic logging
- [ ] Rate limiting
- [ ] Daily backups

### Support Process
- [ ] Escalation workflow documentation
- [x] Onboarding checklist (see [[Data Intake Checklist]])
- [ ] Go-live testing protocol

---

## Later — Do Not Build Yet

- Case studies, testimonials, ROI calculator, slide deck
- BAA template, SOC 2, security audit
- Multi-region, analytics dashboard, self-service portal, auto content refresh
- SLA docs, knowledge base, automated reports, training materials
- Multi-language, voice integration, advanced analytics, A/B testing, API access

---

## Philosophy

> Resist feature creep. The goal is to close one paying client, deliver value, and learn.
> Manual processes are fine for 5-10 clients.
